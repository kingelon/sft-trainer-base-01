Lead Brief Intelligence

From Raw JSON to Actionable Sales Briefs via Fine-tuning

Objective: Demonstrate how fine-tuning transforms a general-purpose LLM into a specialized lead-brief generator — producing consistent, structured summaries from JSON input with a single-line prompt, eliminating fragile prompt engineering.

Why Current Approaches Fall Short

Two approaches reveal architectural limitations — not implementation issues.

Approach 1: Long Prompt Engineering

2000+ token prompts with formatting rules, few-shot examples. Every call carries the full spec.

Works — but fragile, expensive, impossible to maintain at scale. One edit breaks all outputs.

Approach 2: Zero-Shot Base Model

Minimal instruction to base model. Output is verbose, inconsistently structured, variable ordering.

Produces text — but format varies per run, unreliable for automated consumption.
The Insight: A lead brief is a structured extraction and formatting task, not creative generation. The solution must guarantee: Completeness (all fields), Fidelity (numbers match source), Consistency (same structure every run), Readability (optimized for reps).

What Fine-tuning Actually Does

Fine-tuning reshapes the probability distribution over token sequences — making structured output the default behavior, not a prompted suggestion.

Aspect Base Model After Fine-tuning
Knowledge General world knowledge Unchanged — same
Behavior General-purpose, verbose Biased toward trained pattern
Prompt needed 2000+ tokens with examples ~50 tokens, one line
Structure Variable per run Consistent section ordering
Weights changed — 13.6M / 8B (0.17%)

The Self-Distillation Technique

We used Llama 3.1 with a detailed long prompt to generate high-quality lead briefs. These outputs became training data. Fine-tuning then compresses that behavior into a 55MB adapter that triggers with a short instruction.

Key: The adapter learns token-level patterns — which fields first, how to phrase numbers, what structure to follow — not new factual knowledge.

The Value Proposition

⚡

Cost Reduction

10× shorter prompts = proportionally lower API cost per call. At 1000+ daily briefs, significant monthly savings.

🎯

Consistency at Scale

Behavior in the weights, not in a fragile prompt. Model can't "skip" instructions. Same JSON → same structure.

🔧

Operational Simplicity

No prompt versioning, no few-shot management, no prompt injection risk. Deploy adapter → done.

📈

Compounding Quality

Each vetted example permanently improves the model. Unlike prompts where quality resets with each edit.

Why not just keep prompting? Prompt engineering works for prototyping. At scale: (1) costs multiply per call, (2) more instructions = more misinterpretation, (3) prompt maintenance becomes its own burden. Fine-tuning compresses all of that into a 55MB adapter.

Strategic Roadmap

Phase 1: Foundation (Weeks 1–2)

Define Output Specification

Formalize 5-section brief with required fields, ordering rules, acceptance criteria.

Vet Training Data

Audit 200 examples against spec. Remove/correct wrong numbers or missing fields.

Build Automated Evaluation

Field coverage, number fidelity, structure consistency checkers.

Phase 2: Scale (Weeks 3–4)

Expand to 500–1000 Examples

Cover edge cases: missing fields, unusual card counts, varying engagement.

Post-Gen Verification Layer

Cross-reference every number in output against source JSON.

Phase 3: Production (Weeks 5–8)

Structured Output Enforcement

Constrained decoding or JSON-mode to guarantee format compliance.

Preference Tuning (DPO)

Train on good-vs-bad pairs. Directly optimizes the quality gradient.

APPENDIX

How LoRA Fine-tuning Works

The LoRA Mechanism

Freezes all 8B base parameters and injects small trainable matrices into attention and MLP layers.

Base params 8,030,000,000 (frozen)
Adapter params 13,600,000 (trainable)
% trainable 0.17%
Adapter on disk ~55 MB
Training time ~30 min on A100

What Each Layer Controls

Attention (q, k, v, o): What the model attends to — which JSON parts it prioritizes when generating each token.

MLP (gate, up, down): How information transforms — mapping from attended facts to output phrasing.

Training signal: Loss computed only on assistant tokens. Model learns the output pattern, not input echoing.

Rank 16 with all 7 target modules = strong for format learning. Rank 4–8 suffices for simpler style transfer.

Fine-tuning vs. Prompt Engineering

Dimension Prompt Engineering Fine-tuning (LoRA) Winner
Setup cost Low — write prompt, iterate Medium — prepare data, train Prompt
Per-call cost High — 2000+ input tokens Low — 50 input tokens Fine-tuning
Consistency Model can skip/reinterpret Behavior in weights Fine-tuning
Maintenance Prompt versioning, regression Retrain with updated data Fine-tuning
Time to first result Minutes Hours (data + training) Prompt
Quality at 1000+/day Degrades (prompt sensitivity) Stable (weight-based) Fine-tuning
Crossover: Prompts win below ~100 calls/day. Above that, fine-tuning's per-call savings and consistency advantages compound. For lead briefs at enterprise scale, fine-tuning is the economically rational choice.
